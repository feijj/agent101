🐾 OpenClaw 之路

开源免费的 AI Agent 平台，原生支持国产大模型。

🔴 安全警告 — 请认真阅读 OpenClaw 是一个拥有完全文件系统访问权限的开源 AI Agent。它可以读取、修改、甚至删除你电脑上的任何文件。

千万不要在以下环境中使用：
❌ 装有公司源代码的办公电脑
❌ 存有敏感个人资料（银行信息、证件照等）的电脑

✅ 推荐在个人学习电脑或虚拟机 (VM) 中使用。

1 准备运行环境

OpenClaw 需要 Node.js 引擎。脚本会自动检查并配置国内加速镜像。

双击运行：

Windows: tools/windows/1-setup-env.bat

Mac: tools/mac/1-setup-env.command

🍎 Mac 用户 首次运行可能被 Gatekeeper 阻止。请：按住 Control → 点击脚本 → 选择「打开」。

2 安装 OpenClaw

使用脚本一键安装 OpenClaw CLI 工具。脚本会先显示安全警告并请求你确认。

双击运行：

Windows: tools/windows/2-install-openclaw.bat

Mac: tools/mac/2-install-openclaw.command

3 初始化配置 (openclaw onboard)

OpenClaw 有一个交互式的初始化向导，需要在终端中手动运行。这一步无法通过双击脚本完成，需要你打开终端手动操作。别害怕，跟着下面的步骤来就好！

📝 操作步骤

第一步：打开终端

  • Mac: 按 Command + 空格，输入 Terminal，回车
  • Windows: 按 Win + R，输入 cmd，回车

第二步：运行初始化向导

openclaw onboard

第三步：按照向导提示操作

向导会依次询问以下内容，请按照说明作答：

① 选择模型提供商

选择 openai-compatible (OpenAI 兼容模式)。这样可以对接国产大模型。

② 输入 API Base URL

根据你之前申请的大模型平台，输入对应的地址：

# DeepSeek
https://api.deepseek.com/v1

# Kimi（国内版）
https://api.moonshot.cn/v1

# Kimi（海外版）
https://api.moonshot.ai/v1

# 智谱（国内版）
https://open.bigmodel.cn/api/paas/v4

# 智谱（海外版）
https://api.z.ai/api/paas/v4

# MiniMax（国内版）
https://api.minimaxi.chat/v1

# MiniMax（海外版）
https://api.minimax.io/v1

# 通义千问 Qwen（国内版）
https://dashscope.aliyuncs.com/compatible-mode/v1

# 通义千问 Qwen（海外版）
https://dashscope-intl.aliyuncs.com/compatible-mode/v1

③ 输入 API Key

粘贴你之前复制的那串以 sk- 开头的密钥。

④ 选择模型名称

根据你选择的平台输入模型名：

# DeepSeek:               deepseek-chat
# Kimi（国内/海外版）:      kimi-k2.5
# 智谱（国内/海外版）:      glm-5
# MiniMax（国内/海外版）:   MiniMax-M2.5
# Qwen（国内/海外版）:      qwen3-coder-plus

⑤ 选择消息通道

向导会询问你想通过什么方式和 OpenClaw 对话。选择 feishu (飞书)。详细配置见下一步。

4 配置飞书通道 (Feishu)

OpenClaw 需要一个消息通道来接收你的指令并回复你。在国内，我们使用飞书 (Feishu) 作为通道。你需要在飞书开放平台创建一个"机器人应用"。

📝 飞书机器人创建步骤

4.1 注册/登录飞书开放平台

🔗 打开飞书开放平台
如果没有飞书账号，需要先注册一个。

4.2 创建企业自建应用

  • 进入开发者后台 → 点击「创建企业自建应用」
  • 应用名称填写：OpenClaw Bot（随意取名）
  • 应用描述填写：我的 AI 编程助手
  • 点击「创建」

4.3 开启机器人能力

  • 进入刚创建的应用 → 左侧「添加应用能力」
  • 找到「机器人」→ 点击「添加」

4.4 获取 App ID 和 App Secret

  • 左侧「基础信息 → 凭证与基础信息」
  • 复制 App ID 和 App Secret（这两个值很重要！）

4.5 配置权限

  • 左侧「开发配置 → 权限管理 → API 权限」
  • 搜索并添加以下三个权限：
  • im:message.p2p_msg:readonly — 读取用户发给机器人的单聊消息
  • im:message:send_as_bot — 以应用的身份发消息
  • im:message.group_at_msg:readonly — 接收群聊中@机器人消息事件

4.6 配置事件订阅

  • 左侧「开发配置 → 事件与回调 → 事件配置」
  • 订阅模式选择：长连接（不需要公网 IP，最简单！）
  • 点击「添加事件」，搜索并添加：im.message.receive_v1（接收消息）

4.7 发布应用

  • 左侧「应用发布 → 版本管理与发布」
  • 点击「创建版本」→ 设置可用范围 → 点击「申请发布」
  • 等待管理员审批（个人版飞书一般自动通过）

4.8 在 OpenClaw 中配置飞书凭证

打开终端，运行以下命令（替换成你自己的 ID 和 Secret）：

openclaw config set channels.feishu.enabled true
openclaw config set channels.feishu.appId "你的App_ID"
openclaw config set channels.feishu.appSecret "你的App_Secret"

5 启动 Gateway 并开始使用

一切配置完成后，启动 OpenClaw 的 Gateway（网关）守护进程。它会常驻后台，等待你通过飞书发送消息。

启动 Gateway：

openclaw gateway

看到类似 Gateway is running... 的提示后，打开飞书，搜索你创建的机器人名称（如 OpenClaw Bot），发送一条消息：

🎮 试试你的第一个 Prompt

"帮我写一个可以在浏览器里玩的贪吃蛇游戏，界面要现代好看。"
⚠️ Gateway 必须保持运行 关闭终端窗口后，Gateway 会一起关闭，飞书上的机器人也会停止响应。如果想让它一直运行，可以使用 nohup openclaw gateway & 或配合系统的 launchd/systemd 服务。
